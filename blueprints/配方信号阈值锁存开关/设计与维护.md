# 设计与维护

## 核心思路

这版蓝图不再使用“固定延时防抖”，而是采用回差控制：

- 低阈值：决定什么时候可以启动
- 高阈值：决定什么时候可以释放

这样做的目标不是“等几秒再切”，而是“开始补了就先补到一段安全库存再切”。

## 逻辑分段

1. 低阈值减当前库存，得到启动缺口
2. 只保留缺口大于 `0` 的候选
3. 从候选里只选一个，默认取缺口最大的那个
4. 若当前未锁存，则写入新候选
5. 若当前已锁存且对应库存还没到高阈值，则持续保持
6. 一旦达到高阈值，释放锁存，重新选择

## 可调参数

- 低阈值：如 `2000`
- 高阈值：如 `4000`
- 选择策略：当前为“低阈值缺口最大优先”

## 为什么不内置配方映射

有些场景里：

- 监控信号是流体或物品库存
- 机器真正需要的是另一条明确的配方信号

这两者不一定一一对应。  
因此这版蓝图只负责“选中并稳定输出某个原始信号”，把“原始信号 -> 明确配方信号”的映射留给右侧后级逻辑处理。

## 显示器端点

输入端和输出端都改成了 `display-panel`：

- 用文字直接标明接线用途
- 用图标让端点更容易识别
- `always_show = true`，尽量减少只靠 Alt-mode 才能读懂结构的情况
- 说明消息使用 `signal-1 = signal-1` 的恒真条件，不再依赖辅助常量组合器

## 通用脚本

列出已登记蓝图：

```powershell
python .agents/skills/factorio-blueprints/scripts/build_blueprint.py list
```

重建当前蓝图：

```powershell
python .agents/skills/factorio-blueprints/scripts/build_blueprint.py build recipe-signal-switch
```

解码成 JSON 查看：

```powershell
python .agents/skills/factorio-blueprints/scripts/factorio_blueprint_codec.py decode "blueprints/配方信号阈值锁存开关/蓝图.txt" --pretty
```
