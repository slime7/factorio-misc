# 配方信号阈值锁存开关

这个目录采用“蓝图名 = 文件夹名”的组织方式，成品蓝图和说明放在一起。

## 目录内容

- `蓝图.json`：蓝图 JSON 真源，供全局构建脚本编码成蓝图字符串
- `蓝图.txt`：可直接导入 Factorio 的成品蓝图字符串
- `README.md`：用途、接线、使用方式
- `设计与维护.md`：实现思路、脚本入口、可调参数

## 作用

用于从一组库存信号里选出一个当前要补的目标，并以“低阈值启动 + 高阈值释放”的方式锁存输出。

适合场景：

- 同一台设备或同一条后续逻辑同一时刻只能处理一个目标
- 库存在阈值附近波动，直接切换会频繁来回跳
- 希望一旦开始补某种资源，就先补到较高库存再切走

## 输入与输出

- 输入：当前库存信号
- 配置：两个常量组合器，分别填写低阈值和高阈值
- 输出：被锁存的原始信号，输出值固定为 `1`

这个蓝图不做配方信号映射。  
如果某个原料信号需要转换成另一条明确配方信号，请在右侧输出后自行接一层转换逻辑。

## 使用方式

1. 导入同目录下的 `蓝图.txt`
2. 左侧显示器接库存信号来源
3. 右侧显示器接后续转换逻辑或生产设备
4. 在上方常量组合器填写低阈值
5. 在下方常量组合器填写高阈值

## 例子

如果你要监控：

- `熔融铁`
- `熔融铜`

可以设置成：

- 低阈值：两者都填 `2000`
- 高阈值：两者都填 `4000`

这样逻辑就是：

- 某种熔融物低于 `2000` 时，才有资格被选中
- 一旦被选中，通常会继续保持输出
- 如果别的信号缺口明显更大，则可以提前抢占当前锁存
- 如果当前信号一路补到 `4000`，则正常释放并允许切到别的信号
- 如果另一个被监控的信号已经见底，则它可以直接抢占当前锁存

## 注意事项

- 默认策略是“低阈值缺口最大优先，当前锁存只有中等保持补偿，见底信号可强制抢占”
- 多个候选缺口完全相同的话，最终会落到游戏内部的信号排序规则
- 这个“中等保持补偿 + 见底抢占”主要用于处理依赖链场景，例如钢材生产持续消耗熔融铁时，铁的缺口会在见底前后逐步反超钢材
- 右侧输出的是原始信号，不是自动转换后的配方信号
